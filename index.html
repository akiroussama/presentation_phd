CBF

Control Barrier Functions - Theorie

Definition CBF

Fonction h(x): Safe set C = {x : h(x) >= 0}. Garantit que si x(0) in C, alors x(t) in C pour tout t >= 0.

CBF Condition:
dh/dt + alpha(h(x)) >= 0

CBF-QP:
min ||u - u_nom||^2
s.t. Lf*h + Lg*h*u + alpha*h >= 0
Forward Invariance

Systeme ne quitte jamais le safe set - garantie formelle

CBF vs Lyapunov
  • Lyapunov: Stabilite (convergence)
  • CBF: Safety (set invariance)
  • Combinables: CLF-CBF-QP
Avantages CBF
  • Minimal modification de u_nom
  • QP resolvable en temps reel
  • Composable (multi-constraints)
  • Garantie horizon infini

GAPS

Gap Analysis - 12 Gaps Identifies

G1: Safe RL -> HIL

95% simulation only. Aucune methodologie systematique MIL->PHIL.

G2: CBF AC Microgrids

<5 papiers. Non-linearites AC non adressees.

G3: Standards AI/RL

IEEE 1547 ignore AI. Comment certifier un agent RL?

G4: DT Real-Time Safety

DT pour monitoring OK. Safety guarantees absentes.

G5: Islanding + CBF

Detection ML rapide, mais pas de CBF-aware controller.

G6: GFM + Safe RL

Grid-forming critique, mais pas d'integration RL safe.

GAPS

Gaps 7-12 - Validation & Future

G7: Multi-Timescale CBF

CBF pour un niveau. Integration primaire-secondaire-tertiaire absente.

G8: Uncertainty CBF

CBF deterministe. Robust CBF pour stochastic peu explore.

G9: Scalability

CBF-QP pour 100+ DERs? Decentralisation necessaire.

G10: Communication

CBF assume info instantanee. Delays et packet loss ignores.

G11: Market Integration

CBF pour safety physique. Integration marche (pricing) absente.

G12: Cyber-Physical

Cyber attacks sur CBF? Resilience non etudiee.

Gap Critique pour PhD

G1 + G2: Safe RL avec CBF pour AC microgrids, valide sur HIL. C'est le coeur de l'opportunite de recherche.
